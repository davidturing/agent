# 华为数据之道 - 第2章：建立企业级数据综合治理体系

数据作为一种新的生产要素，在企业构筑竞争优势的过程中起着重要作用，企业应将数据作为一种**战略资产**进行管理。华为公司经过十多年的实践证明，只有构筑一套企业级的**数据综合治理体系**，才能确保关键数据资产有清晰的业务管理责任，IT建设有稳定的原则和依据，作业人员有规范的流程和指导；当面临临争议时，有裁决机构和升级处理机制；治理过程所需的人才、组织、预算有充足的保障。

### 架构图解析：图 2-1 华为数据治理体系框架

华为的**数据治理体系框架**是一个综合性的结构，旨在确保数据的质量和安全得到保障，使数据的价值能真正发挥出来。该框架自上而下由政策、流程、组织和IT平台四个层面构成，并由数据架构标准和数据质量管理两大支柱纵向贯穿。

*   **政策指引 (顶层)**：
    *   核心是**公司数据管理总纲**，作为所有数据管理活动的最高指导原则。
    *   总纲下设三大核心政策：
        1.  **信息架构管理政策**
        2.  **数据质量管理政策**
        3.  **数据源管理政策**
    *   _这些政策共同构成了数据治理的顶层设计，明确了方向和原则。_

*   **流程 (中间层)**：
    *   定义了数据管理的核心活动流程，包括：
        1.  **管理信息架构**
        2.  **管理数据质量**
        3.  **管理数据分析**
    *   这些流程将顶层政策转化为可执行的行动。

*   **组织 (中间层)**：
    *   明确了数据治理的责任主体和组织架构，包括：
        1.  **公司数据Owner**：最高决策者。
        2.  **数据管理部**：专业的治理执行与支持组织。
        3.  **信息架构专家组**：提供专业评审和决策支持。

*   **IT (底层)**：
    *   提供支撑数据治理流程和组织运作的技术平台与工具，包括：
        1.  **主数据管理平台**
        2.  **维度数据管理平台**
        3.  **元数据管理平台**
        4.  **数据质量管理工具**

*   **两大支柱 (纵向贯穿)**：
    *   **数据架构标准管理**：确保数据语言的统一和互通。
    *   **数据质量管理**：通过持续的监控和改进，提升数据可信度。

## 2.1 建立公司级的数据治理政策

**数据治理政策**是华为数据治理的顶层设计，在华为公司由EMT（经营管理团队）汇报通过，由总裁签发。该政策明确了数据工作在公司治理体系中的地位，体现了公司管理层对数据工作重要性的统一认知。

### 2.1.1 华为数据管理总纲

**华为数据管理总纲**明确了数据管理最基本的原则，包括信息架构、数据产生、数据应用及数据质量的职责和分工等，确保数据治理环境的有效构建。

1.  **(1) 信息架构管理原则**
    *   **第一条**：建立企业级**信息架构**，统一数据语言。
    *   **第二条**：所有变革项目须遵从数据管控要求。对于不遵从管控要求的变革项目，数据管控组织拥有一票否决权。
    *   **第三条**：应用系统设计和开发应遵从企业级信息架构。关键应用系统必须通过应用系统认证。

2.  **(2) 数据产生管理原则**
    *   **第一条**：数据规划对齐业务战略，业务战略规划必须包含关键数据举措及其路标规划。
    *   **第二条**：公司**数据Owner**拥有公司数据的管理最高决策权，依托ESC（变革指导委员会）决策平台议事。各数据Owner承担数据工作路标、信息架构、数据责任机制和数据质量的管理责任。
    *   **第三条**：关键数据须定义单一数据源，一点录入，多点调用。_数据质量问题应在源头解决。_
    *   **第四条**：谁产生数据，谁对数据质量负责。数据Owner负责基于使用要求制定数据质量标准，且须征得关键使用部门的同意。

3.  **(3) 数据应用管理原则**
    *   **第一条**：数据应在满足信息安全的前提下充分共享，数据产生部门不得拒绝跨领域的、合理的数据共享需求。
    *   **第二条**：信息披露、数据安全管理、数据保管和个人数据隐私保护等必须遵守法律法规和道德规范的要求。公司保护员工、客户、商业伙伴和其他可识别个体的个人数据。

4.  **(4) 数据问责与奖惩管理原则**
    *   各数据Owner应建立数据问题回溯和奖惩机制。对不遵从信息架构或存在严重数据质量问题的责任人进行问责。

### 2.1.2 信息架构管理政策

**信息架构**是公司统一的数据语言，是业务流打通、消除信息孤岛和提升业务流集成效率的关键要素。

1.  **(1) 管理信息架构的角色与职责**
    *   **第一条**：公司数据Owner负责批准企业级信息架构，裁决重大信息架构问题和争议。
    *   **第二条**：各数据Owner负责其所辖数据的信息架构建设和维护，承接及落实公司的数据规划要求。
    *   **第三条**：公司的数据管理专业组织作为公司数据工作的支撑组织，负责组织信息架构的建设、维护、落地及遵从管控，负责协调跨领域的信息架构冲突。
    *   **第四条**：数据管控组织作为信息架构专业评审机构，确保信息架构的质量和集成。

2.  **(2) 信息架构建设要求**
    *   **第一条**：关键数据应被识别、分类、定义及标准化，数据的定义在公司范围内应唯一，数据标准制定要考虑跨流程要求。
    *   **第二条**：数据资产目录必须承接公司各业务环节的使用需求和报告分析最小粒度的要求。
    *   **第三条**：信息架构驱动应用架构设计，合理规划数据分布。
    *   **第四条**：应用系统数据库的设计和开发要遵循信息架构，减少数据冗余，实现接口标准化。

3.  **(3) 信息架构遵从管控**
    *   **第一条**：变革项目必须遵从已发布的信息架构，变革项目的交付件须包含信息架构内容。
    *   **第二条**：业务流程设计必须遵从已发布的信息架构，在流程说明文件、操作指导书或模板类文件中体现。
    *   **第三条**：应用系统设计必须遵从已发布的信息架构。_对于不满足要求的应用架构交付件和应用系统设计交付件，不予上线。_

### 2.1.3 数据源管理政策

**数据源**是华为数据治理的核心观点之一。数据源是指业务上首次正式发布某项数据的应用系统，经过数据管理专业组织认证，作为唯一数据源被周边系统调用。

1.  **(1) 数据源管理原则**
    *   **第一条**：所有关键数据必须认证数据源。关键数据是指影响公司经营、运营报告的数据，在公司范围内统一发布。
    *   **第二条**：数据管理专业组织为关键数据指定源头，数据源必须遵从信息架构和标准，经信息架构专家委员会认证后成为数据源。
    *   **第三条**：所有关键数据仅能在数据源录入、修改，全流程共享，其他调用系统不能修改。
    *   **第四条**：所有应用系统必须从数据源或数据源镜像获取关键数据。
    *   **第五条**：数据Owner确保数据源的数据质量，对不符合数据质量标准的，必须限期整改。

2.  **(2) 数据源认证标准**
    *   **第一条**：数据源是在信息链上正式发布的第一个数据存储系统。
    *   **第二条**：数据源是某项数据唯一的录入点。
    *   **第三条**：数据源必须是数据维护最为及时、正确、完整的数据存储系统。
    *   **第四条**：数据源所在系统的性能和可用性应当满足其他调用系统的数据访问需求。

### 2.1.4 数据质量管理政策

**数据质量**的持续提升是华为数据治理的核心目标。

1.  **(1) 数据质量管理职责及要求**
    *   **第一条**：各数据Owner负责保障所辖数据的质量，承接公司数据Owner设定的数据质量目标，制定数据质量标准及测评指标，持续度量与改进。
    *   **第二条**：公司全员在业务执行的过程中应确保业务记录满足数据质量要求。
    *   **第三条**：财经各级CFO组织应遵循职业道德准则，诚实记录和报告财经数据，承担财务监控和及时报告责任。
    *   **第四条**：公司各级数据管理专业组织为数据Owner提供数据质量管理专业支撑。
    *   **第五条**：内控组织应将数据质量管控要素的执行情况纳入**SACA（Semi-Annual Control Assessment，半年度控制评估）**评估范围，推动数据质量问题的闭环管理。
    *   **第六条**：内审部门作为独立机构，负责重大数据问题的审计和责任回溯。

2.  **(2) 数据质量的业务规则和管理要求**
    *   **第一条**：流程建设应考虑数据质量要求，将数据的关键质量控制要素纳入关键控制点。
    *   **第二条**：数据Owner负责基于使用要求制定数据质量标准，且须征得关键使用部门的同意。
    *   **第三条**：数据创建应确保录入正确，关键数据应进行复核或审批。
    *   **第四条**：对影响关键经营指标的数据造假行为采取零容忍态度。
    *   **第五条**：上游环节应保证企业数据的真实、完整并及时传递到下游环节。
    *   **第六条**：因外部原因频繁变化的基础数据，数据Owner应及时维护并统一发布最新数据。
    *   **第七条**：数据质量应持续进行度量。
    *   **第八条**：报告与分析的数据层级和最小粒度应适度，能与最小业务信息单元相匹配。

## 2.2 融入变革、运营与IT的数据治理

华为公司依托变革管理体系，进行流程、数据与应用系统建设，同时持续优化运营管理体系。数据从业务中产生，在IT系统中落地，决定了数据治理工作必须充分融入业务运营与IT系统建设中。

### 2.2.1 建立管理数据流程

为了支撑企业数据从架构设计、质量管理到数据分析应用的全生命周期管理，需要建立一个**管理数据流程**。

*   **流程定位**：华为将“**管理数据**”流程定位为“**管理BT&IT**”流程下的一个L2流程。
*   **流程分解**：该L2流程下设三个L3子流程：
    1.  **管理信息架构**
    2.  **管理数据质量**
    3.  **管理数据分析**
*   **关键角色与职责**：
    *   **信息架构工程师**：负责数据架构设计和管控，数据分类、定义及标准化，企业级信息架构、业务侧概念模型开发和维护等。
    *   **数据治理工程师**：负责聚焦数据资产建设和治理，数据治理和数据质量监控，识别和定位数据质量问题等。
    *   **数据平台工程师**：负责数据分析平台规划和运营，数据采集和预处理。
    *   **数据分析师**：负责聚焦价值实现，数据分析和挖掘，业务数据模型开发，数据可视化设计等。
    *   **数据科学家**：负责聚焦技术研究和攻关，基础数据模型和算法的开发，数据产品设计，数据分析问题攻关。

### 2.2.2 管理数据流程与管理变革项目、管理质量与运营之间的关系

*   **流程图解析：图 2-3**
    *   该图展示了一个双向循环关系。
    *   **左循环 (管理变革项目)**：业务方案驱动数据解决方案，数据解决方案驱动业务方案的优化。
    *   **右循环 (管理质量与运营)**：业务方案驱动数据解决方案，数据解决方案支撑业务方案的落地和运营。
    *   **核心环节**：**管理数据**流程位于中间，负责数据解决方案的开发与验证，其产出包括**管理信息架构**、**管理数据质量**和**管理数据分析**。这表明数据治理是连接业务变革与日常运营的桥梁。

### 2.2.3 通过变革体系和运营体系进行决策

*   **决策体系解析：图 2-4 华为数据治理决策体系**
    *   这是一个分层的决策模型，确保数据治理的决策能够高效、专业地进行。
    *   **顶层决策机构**：
        *   **变革指导委员会**：公司最高决策机构。
        *   **各业务领域全球流程责任人**：负责各自领域内的流程和数据。
    *   **专业评审与管理机构**：
        *   **变革管理体系**：
            *   **变革管理办公室/企业架构委员会**：负责整体变革管理。
            *   **业务架构/信息架构/应用架构/技术架构专家组**：提供各领域的专业评审。
        *   **流程运营体系**：
            *   **业务运作团队/质量运营团队**：负责日常运营。
            *   **数据管家**：负责具体的数据管理执行。
    *   **执行层**：
        *   **变革项目**：在变革管理体系下执行。
        *   **业务优化/数据攻关团队**：在流程运营体系下执行。

### 2.2.4 数据治理融入IT实施

业务人员通过IT产品提供的功能和服务提升作业效率。因此，对业务数据的管理要求，必然要落实到IT产品的操作界面和数据库设计中。

*   **实践要求**：
    *   在IT产品团队中设置**系统架构师**和**数据架构师**角色，负责界面设计、数据库设计、数据集成方案设计，向上承接信息架构的设计要求。
    *   明确界面里的字段要遵从数据标准的定义，数据库表和字段的设计要承接信息架构的设计要求，从而达到数据治理融入IT实施流程的目标。

### 2.2.5 通过内控体系赋能数据治理

要对华为这样的大型企业实施数据治理是非常复杂的事情。仅仅通过数据管理部门对各个项目和部门的培训、指导、人员支持，不足以确保公司的治理目标和要求有效地贯彻到位。

*   **实践方法**：
    *   华为通过**内控体系**，每年实施**SACA评估**和数据专项内部审计，揭示数据治理过程的问题，确定改进目标和责任人，从而保证数据治理机制的有效运作。

## 2.3 建立业务负责制的数据管理责任体系

业务即行为，行为即记录，记录即数据。华为的每一个数据，必须由对应的业务部门承担管理责任，且必须有唯一的数据Owner。**业务负责制**的数据管理责任体系，是华为数据治理体系多年实践经验的结晶，是确保体系发挥作用的基石。

### 2.3.1 任命数据Owner和数据管家

华为按分层分级原则任命**数据Owner**，在公司层面设置公司数据Owner，在各业务领域设置领域数据Owner，这样既能确保公司数据工作统筹规划，也能同时兼顾各业务领域灵活多变的特征。

*   **公司数据Owner职责**：
    1.  制定数据管理体系的愿景和路标。
    2.  传播数据管理理念，营造数据文化氛围。
    3.  建设和优化数据管理体系，包括组织与任命、授权与问责等。
    4.  批准公司数据管理的政策和法规。
    5.  裁决跨领域的数据管理及管理争议，解决跨领域的重大数据及管理问题。

*   **各级流程Owner职责**（即该领域的**数据Owner**）：
    1.  负责数据管理体系建设。
    2.  负责信息架构建设。
    3.  负责数据质量管理。
    4.  负责数据底座和数据服务建设。
    5.  负责数据争议裁决。

*   **数据管家 (Data Steward)**：
    *   是数据Owner的助手，是数据Owner在数据管理方面的具体执行者。

### 2.3.2 建立公司层面的数据管理组织

*   **组织架构解析：图 2-5 华为数据管理组织**
    *   **核心管理层**：
        *   **公司数据Owner** 和 **质量与流程IT管理部** 处于顶层。
    *   **领域管理层**：
        *   **领域数据Owner** 和 **质量运营部/XX管理部** 负责各业务领域的数据治理。
    *   **执行团队**：
        *   **跨领域数据联合集成团队**：由各领域的数据管理部人员组成，负责执行具体的数据治理任务，如数据质量执行组、信息架构建设组、数据服务推进组等。
    *   **能力中心 (左侧)**：
        *   提供体系建设、组织定位和文化倡导等方面的能力支持。

*   **数据管理组织的四类职责**：
    1.  **1) 体系建设者**：负责数据管理的战略、规划、政策、规则的制定。
    2.  **2) 能力中心**：构建数据管理的方法、工具、平台；负责专业能力的开发和建设。
    3.  **3) 业务的数据伙伴**：面向业务，提供数据解决方案，解决业务数据痛点；支撑业务数据需求。
    4.  **4) 文化倡导者**：在公司范围追求卓越，“谁创建（录入）数据，谁对数据质量负责”的文化；用数据支撑业务决策的文化。

该图展示了华为为数据全生命周期治理建立的规范体系，覆盖了从业务感知到IT开发再到数据消费的全过程。
