# 8. 打造“清洁数据”的质量综合管理能力

_越来越多的企业应用和服务都基于数据而建，数据质量是数据价值得以发挥的前提。例如企业运营效率主要依赖于数据获取的准确性和及时性，企业客户关系管理系统中的错误或不完整数据将导致客户沟通不顺畅，影响客户满意度。_

随着数据类型、数据来源的不断丰富以及数据量的飞速增长，企业面临的数据质量问题的概率显著增加。**数据质量**是一个复杂问题，往往是多种因素综合作用的结果，解决**数据质量**问题要从机制、制度、流程、工具、管理等多个方面发力。

本章讲述**数据质量**基本概念、管理框架，详细说明**数据质量**控制、**数据质量**改进、**数据质量**度量的基本方法。

## 8.1 基于 PDCA 的数据质量框架

企业数据来源于多个不同的业务系统，数据流转、处理环节多，用“**Garbage in Garbage out**”（垃圾进，垃圾出）原则保证**数据质量**已成为数字化转型企业的共识。企业**数据质量**管理是一个系统性的工程，华为数据质量从**数据质量**领导力、**数据质量**持续改进、**数据质量**能力保障三方面展开，有机结合形成联动。

### 8.1.1 什么是数据质量

*   **ISO9000 标准定义**：**数据质量**指“产品固有特性满足要求的程度”，其中“要求”指“明示的、隐含的或必须履行的需求或期望”，强调“以顾客为关注焦点”。
*   **学术界定义**：在 Won Kim 的论文“A Taxonomy of Dirty Data”中，**数据质量**被定义为“**适用性 (fitness for use)**”，即数据适合使用的程度、满足特定用户期望的程度。
*   **华为定义**：**数据质量**指“数据满足应用的**可信程度**”，并从以下六个维度对**数据质量**进行描述：
    1.  **完整性 (Completeness)**：指数据在创建、传递过程中无缺失和遗漏，包括实体完整、属性完整、记录完整和字段值完整四个方面。_完整性是数据质量最基础的一项，例如员工工号不可为空。_
    2.  **及时性 (Timeliness)**：指及时记录和传递相关数据，满足业务对信息获取的时间要求。数据交付时间过长可能导致分析结论失去参考意义。
    3.  **准确性 (Accuracy)**：指真实、准确地记录原始数据，无虚假数据及信息。数据要准确地反映其所建模的“真实世界”实体。例如员工的身份信息必须与身份证件上的信息保持一致。
    4.  **一致性 (Consistency)**：指遵循统一的数据标准记录和传递数据和信息，主要体现在数据是否符合规范。例如同一个员工号对应的员工姓名需一致。
    5.  **唯一性 (Uniqueness)**：指同一个数据只能有唯一的标识符。体现在一个数据集中，一个实体只出现一次，并且每个唯一实体有一个键值且该键值只指向该实体。例如一个员工有且仅有一个有效工号。
    6.  **有效性 (Validity)**：指数据的值、格式和展现形式符合数据定义和业务定义的要求。例如员工的国籍必须是国家基础数据中定义的值。

### 8.1.2 数据质量管理范围

在提及**数据质量**管理时，经常有人会问：**数据质量**和**流程质量**有什么区别？
*   **流程质量**是基于流程结果评估业务执行的好坏。
*   **数据质量**更关注业务对象、业务规则、业务过程、业务结果等数据是否得到了及时记录。

_以采购验收为例，采购验收及时性属于流程质量合格，而验收数据录入及时性属于数据质量，验收录入人所需时间满足1天的SLA即属于数据质量合格。_

### 8.1.3 数据质量的总体框架

华为以 **ISO8000 质量标准体系**为依据，设计了 **PDCA (Plan、Do、Check、Action，计划、执行、检查、处理)** 持续改进的**数据质量**管理框架。

该框架（图 8-1）由三个核心部分构成：
1.  **领导力 (Leadership)**：
    *   **数据质量政策**：制定公司级的**数据质量**方针。
    *   **数据质量管控**：建立**数据质量**的管控机制。
    *   **数据质量文化**：营造全员关注**数据质量**的文化氛围。
2.  **持续改进 (Continuous Improvement)**：
    *   这是框架的核心，是一个由业务需求驱动的 **PDCA** 循环。
    *   **计划 (Plan)**：进行**数据质量**策划 (SP/BP)。
    *   **执行 (Do)**：实施**数据质量**控制和**数据质量**改进。
    *   **检查 (Check)**：进行**数据质量**度量。
    *   _整个循环的目标是提升结果，即干系人满意。_
3.  **能力保障 (Capability Guarantee)**：
    *   **数据组织**：建立负责**数据质量**的组织和角色。
    *   **数据流程**：规范**数据质量**管理的相关流程。
    *   **IT 平台**：提供支撑**数据质量**管理的工具和平台。

**数据质量**管理以数据清洁为目标，以业务需求为驱动，通过 **PDCA** 的循环，提升**数据质量**，达到**数据质量**结果满意。领导力模块通过制定政策、规范来构建**数据质量**机制，对**数据质量**的工作起到牵引作用。能力保障模块构建完整的**数据组织**、流程和工具，起到支撑作用。

## 8.2 全面监控企业业务异常数据

不论做了多少**数据质量**预防措施，实施多严格的**数据质量**过程控制，只要涉及人为干预，总会存在**数据质量**的问题。为了避免或降低低质量数据对业务的影响，要能及时发现**数据质量**问题。问题的发现既可以“**正向**”主动监控，也可以“**逆向**”通过下游环节反馈问题来识别。_主动发现、制定解决方案、采取行动，比被动采取补救措施效果更好，并且代价更小。_

### 8.2.1 数据质量规则

**数据质量规则**是判断数据是否符合**数据质量**要求的逻辑约束。在整个数据监控的过程中，好的坏直接影响监控的效果，因此如何设计**数据质量**规则很重要。

依据数据在库落地时的质量特性及**数据质量**规则类型，华为设计了如下四类**数据质量**规则框架：
1.  **单列数据质量规则**：关注数据属性值的有无以及是否合自身规范的逻辑判断。
2.  **跨列数据质量规则**：关注数据属性间关联关系的逻辑判断。
3.  **跨行数据质量规则**：关注数据记录之间关联关系的逻辑判断。
4.  **跨表数据质量规则**：关注数据集关联关系的逻辑判断。

华为结合 **ISO8000 数据质量标准**、数据质量控制与评估原则（国标 SY/T 7005—2014），共设计了15类规则，具体如表 8-1 所示：

| 业务对象落地 | 质量特性 | 规则类型 | 类型描述 | 示例 |
| :--- | :--- | :--- | :--- | :--- |
| 单列 | 完整性 | 不可为空类 | 属性值允许在满足条件时不允许出现空值 | 员工工号不可为空 |
| | 有效性 | 语法约束类 | 属性值需满足数据语法或规范政策约束 | 邮箱地址需满足国家通信管理局的邮箱格式，身份证号码满足国家标准 |
| | 有效性 | 格式规范类 | 属性值必须满足属性格式约束 | 日期有多重格式，对于同一属性指定一种格式 |
| | 有效性 | 长度约束类 | 属性值须满足约定的长度范围 | 密码的长度至少要8位，不超过16位 |
| | 有效性 | 值域约束类 | 属性值必须满足已定义的枚举值列表的约束 | 合同的合同类型必须是合同基础数据中定义的枚举值 |
| 跨列 | 准确性 | 事实参照核对类 | 存在事实数据或事实参照事件参考标准数据，与该事实或事实参照标准对比一致的约束 | 中国通信有限公司的信息必须与国家法人数据库中的信息保持一致 |
| | 完整性 | 应有值类 | 敏感站点不允许维护维护信息 | |
| | 一致性 | 单表等值一致约束类 | 某一属性值与实体其他属性值满足逻辑关系 | 合同的签约金额必须等于合同下所有产品报价的总和 |
| 跨行 | 一致性 | 单表映射一致性约束 | 某一属性值与本实体其他属性值满足逻辑关系 | 合同的关闭日期不能早于注册日期 |
| | 及时性 | 入库及时类 | 数据进入系统的时间及时性约束，通常包括数据源数据获取时间和入库时间才能进行检测 | 通过 HRMS 系统中员工的入职日期和系统创建日期判断员工入职信息维护及时性 |
| | 一致性 | 外关联约束类 | 引用其他业务对象属性时，所维护的属性必须在其他业务对象中存在的约束 | 合同的签约客户必须为客户主数据中定义的法人客户 |
| | 一致性 | 跨表等值一致性约束 | 某一属性值与其他实体的多个属性值的统计结果相等的约束 | 合同的金额与合同按产品报价拆分的金额之和一致 |
| | 一致性 | 跨表映射一致性约束 | 某一属性值满足其他实体的多个属性值的函数关系的约束 | 员工的任命日期必须早于员工的到岗日期 |
| 跨行 | 唯一性 | 记录唯一类 | 记录不重复，存在可识别业务的业务主键，数据源内部不存在行级或复杂记录的约束 | 法人客户在中国移动通信有限公司只能唯一存在一笔 |
| | 一致性 | 层级结构一致性约束 | 存在连续结构的属性，同层级属性结构一致 | 所有子网段的客户，满足总部-子网-子网的三层结构 |

**华为实践：规则树**
为了系统化地管理和应用规则，华为提出了“**规则树**”的概念。以“**邮箱地址**”为例，对其质量检查涉及多个规则：
*   **不可为空类**：邮箱地址不能为空。
*   **语法约束类**：地址中必须包含“@”。
*   **格式规范类**：中国区需以“.cn”结尾。

这三个子规则向上收敛，形成“**邮箱地址**”业务属性的完整主规则。这种层级关系我们称之为“**规则树**”。通过**规则树**，既能统计出共有多少员工的“邮箱地址”数据异常，又可分别识别出每个子规则的异常数量，从而快速识别出当前哪个问题更严重，以便优先解决。

### 8.2.2 异常数据监控

**质量控制**是通过监控**数据质量**形成过程，消除全过程中引起不合格或不满意效果的因素，以达到质量要求而采用的各种质量作业技术和活动。

华为通过**数据质量**监控平台，以异常数据管理为核心，实施**数据质量**控制。其核心步骤如下：
1.  **识别监控对象范围，确定监控内容**
    *   **数据质量**控制从明确业务需求开始，根据业务规则和数据相关的需求，阶段性确定数据质量控制范围。
    *   **重要性原则**：
        *   **关键主数据和基础数据**：公司级、领域级主数据，如产品、客户、供应商、组织、人员、站点。
        *   **关键的事务数据**：主交易流的核心事务数据，如客户合同、BOQ、工程服务采购 PR、S&OP 计划、采购 PO。
        *   **痛点问题**：领域业务运营痛点问题、公司级变革、攻关项目、业务核心 KPI 等涉及的对象纳入度量，如产品 Item。
    *   **成本效益原则**：
        *   运作成熟且质量较高的数据，或度量成本很高但预期的改进很少的数据，可不优先考虑。
2.  **数据源剖析**
    *   在着手设计**数据质量**规则前，需对数据进行快速数据剖析，目的是分析数据源的内容、质量和结构，同时发现和分析数据源中所有数据不规范问题和使数据项目处于危险中的隐藏数据问题。
    *   **数据源内容**：通过可视化视图，可以了解表中包含员工工号、姓名等内容，即列信息。
    *   **数据源结构**：包括技术结构和业务结构。技术结构指空值频率、相异值频率、值范围、模式、长度、数据类型等。业务结构如组织结构存储是平面结构还是树状结构。
    *   **数据源质量**：根据数据标准分析剖析结果的数据质量，例如必填字段是否有空值存储，有允许值列表的值个数与相异值频率是否一致等。
3.  **设计和配置监控规则，自动监测异常数据**
    *   华为**数据质量**平台已实现**数据规则**的可配置、数字化、快速部署、自动监控识别异常数据等能力，并可通过虚拟化的方式快速、灵活发布监控结果。
    *   **自动化监控流程**：
        *   **配置规则**：在 IT 工具中配置**数据质量**规则。
        *   **自动监控**：系统自动执行规则，识别异常数据。
        *   **发布监控结果**：通过虚拟化的方式快速发布结果（如异常明细、分级数据表），供业务人员定位和分析问题。

## 8.3 通过数据质量综合水平牵引质量提升

通过**数据质量**度量综合评价公司整体**数据质量**水平，制定**数据质量**基线，披露**数据质量**问题与短板，促进问题改进，推动数据 Owner 承接**数据质量**改进目标，持续提升**数据质量**，实现数据清洁。

### 8.3.1 数据质量度量运作机制

**数据质量度量模型**
华为的度量模型（图 8-10）将过程设计与执行结果并重，其核心思想是：
*   **总分 = 设计质量 (40%) + 执行质量 (60%)**
*   **设计质量**：评估信息架构的建设情况，包括**数据资产目录**、**数据标准**、**数据模型**、**数据分布**四个方面。
*   **执行质量**：评估数据内容的清洁度，聚焦影响“财报”和“业务运营”的关键数据，从**完整性**、**及时性**、**准确性**、**一致性**、**唯一性**、**有效性**六个维度进行评估。

**运作机制**
*   **角色职责**：
    *   **公司数据管理部**：根据公司**数据质量**管理工作规划，制定**数据质量**目标；组织**数据质量**度量工作开展，发布公司**数据质量**度量报告。
    *   **各领域数据管理部**：拟定**数据质量**标准并设计指标，执行**数据质量**度量；组织各领域业务专家，分析**数据质量**问题根因，制定改进举措及闭环管理。
*   **度量规则**：
    *   **度量对象选定原则**：聚焦业务运营痛点和影响财报的关键数据。
    *   **度量频率**：一年度量两次。上半年（1~6月）重点监控质量改进状况；全年度量（1~12月）综合评价质量达成水平。
    *   **评价标准**：统一采取百分率的方式评价，并根据度量得分划分如表 8-2 所示的五档。

| 评分标准 (百分率) | 定级评价 (5等级) |
| :--- | :--- |
| 80% ~ 100% | 1等 (满意) |
| 60% ~ 80% | 2等 (基本满意) |
| 40% ~ 60% | 3等 (略不满意) |
| 20% ~ 40% | 4等 (不满意) |
| 0 ~ 20% | 5等 (很不满意) |

### 8.3.2 设计质量度量

为确保**数据质量**标准稳定，从信息架构的四个角度（**数据资产目录**、**数据标准**、**数据模型**、**数据分布**）进行综合评估。其范围覆盖度量期间内已通过 IA-SAG 评审发布的所有数据资产。

**质量打分细则（图 8-11）**
*   **1分 (差)**：业务对象未定义定责及 Owner；未发布 IA-SAG 评审后的概念模型；未认证数据源。
*   **2分 (中)**：业务对象有明确的定责及 Owner；开发完成概念模型，并通过 IA-SAG 评审；数据源认证通过 IA-SAG 评审。
*   **3分 (良)**：数据标准覆盖 GPO、BPO 或 GPC 签署；数据标准在 IT 系统及对应业务流程中得到应用遵从；明确数据模型物理落地关系，并制定出稳定的整改方案及路标。
*   **4-5分 (优/满分)**：完成数据模型物理落地及数据字典发布；完成数据血缘信息梳理，确保数据信息透明，消除人工干预。

### 8.3.3 执行质量度量

**执行质量**度量主要是从**数据质量**六性（一致性、完整性、及时性、唯一性、有效性、准确性）评估数据内容的清洁度，涉及三个要素：**客户关注重要性**、**法律财务风险性**、**业务流程战略性**。

**1. 确定度量指标**
*   **数据质量**度量指标也往往来源于日常监控的**数据质量**规则，将业务属性层规则通过叠加公式变成业务对象层度量指标。
*   **指标设置原则**：
    *   **重要性原则**：对核心数据、痛点问题较严重的数据，需考虑设计度量指标。
    *   **成本效益原则**：运作成熟且质量较高的数据，或度量成本很高但预期改进很少的数据，可以考虑简化度量指标或不度量。
    *   **明确性原则**：指标设计清晰、可衡量。
    *   **分层分级原则**：根据不同层级的管理诉求，设计分层分级的指标。
    *   **持续度量原则**：一次性就可解决问题的数据不需要度量。
*   **计算公式**：
    *   **数据面积算法**：`逻辑实体数据质量度量指标 = ∑属性数据质量异常数量 / ∑属性数据总量`。
    *   **平均算法**：`业务对象数据质量度量综合指标 = Average(逻辑实体数据质量度量指标)`。
    *   _不直接在业务对象层采用数据面积算法，是为了避免重要的错误数据被“淹没”。_

**2. 确定数据质量衡量标准**
*   **数据质量**衡量标准是指南测评结果与用户诉求的关系。华为主要采用五个等级（差、中、良、优、满分）来衡量和拉通数据质量满足消费者的应用程度。
*   _示例一：企业供应商数量为2000家，其中供应商账号信息中的“收款账号”信息准确率为90%，即有200位供应商的“付款账号”数据错误，这意味着有10%的应付可能出现付款错误。对于这个准确率数据，消费者是不能接受的。_
*   _示例二：企业度量“员工现居住地准确率”结果为90%。调研统计结果表明有50%的员工处于租房状态，因此数据消费者认为当前的数据质量可以满足他们的应用要求。_

### 8.3.4 质量改进

**数据质量**改进致力于增强满足**数据质量**要求的能力。**数据质量**改进的步骤本身就是一个 **PDCA** 循环。
*   **数据质量控制与改进的关系**：
    *   **质量控制**：维持现有的**数据质量**水平，其方法是数据质量控制；改善是指改进目前的**数据质量**。
    *   **质量改进**：方法是主动采取措施，使**数据质量**在原有的基础上有突破性的提高。
*   **质量控制是质量改进的前提**，控制就意味着维持以前的质量水平，是 PDCA 改进循环中保证水平不下降的“努力的楔子”，是保证下一次改进的起点，而改进则是在起点基础上的变革和突破。

## 8.4 本章小结

**数据质量**管理应成为企业持续、例行的工作，企业**数据质量**管理水平直接影响数据应用的效率和数字化转型的成效。华为**数据质量**管理框架由三个部分构成，包括自上而下打造**数据质量**领导力、全面推进**数据质量**持续改进机制、不断加强**数据质量**能力保障。通过制定**数据质量**政策，并依托公司变革体系和流程运营体系实现**数据质量**的落地，同时以多种方式在全公司营造**数据质量**氛围和文化。其中最重要的是建立了企业数据质量改进的机制，即基于**数据质量**的 **PDCA** 循环——数据质量策划、控制、度量和改进。最后通过组织、流程、IT 三个方面的能力保障，使**数据质量**管理“**系统化**”“**持续化**”“**常态化**”。
