# 4.1 数据架构设计的核心逻辑与实践范式

> **摘要**: 数据架构是企业数字化的骨骼。本节深度剖析两大标杆——华为的“三层分层解耦”架构与阿里的“五层中台化”架构。这两大流派分别代表了“严谨工程派”与“敏捷互联网派”的最高水平，也是企业实战中最常参考的模板。

---

## 4.1.1 华为数据架构设计与落地：业务驱动的“三层解耦”

华为的数据架构深受 IBM 和 Inmon 理论影响，强调逻辑严密、分层解耦和资产沉淀。

### 1. 核心设计理念
*   **分层解耦 (Decoupling)**: 华为认为，数据产生系统（源头）和数据消费系统（应用）必须解耦。若直接从 ERP 抽数给 BI，ERP 升级会导致 BI 挂掉。因此，必须建立稳定的中间层。
*   **一书一表**: 
        *   **数据字典 (一书)**: 业务元数据的圣经，统一定义所有业务术语。
        *   **数据地图 (一表)**: 全局数据资产的导航图。

### 2. 三层架构详解

#### 第一层：源数据层 (Source Layer)
*   **功能**: 镜像接入全量企业数据。
*   **策略**: **Raw Data (原样保留)**。除了基础的去重和格式化一律不改，确保能回溯到业务发生的原始现场。
*   **覆盖**: 包含 ERP (结构化)、PLM (工程文档)、行政 IoT (传感器) 等多源异构数据。

#### 第二层：数据集成层 (Integration/Foundation Layer) —— **核心**
*   **功能**: 清洗、转换、标准化。这是治理最重的地方。
*   **架构**: 湖仓一体 (FusionInsight)。
*   **动作**: 
        *   **主数据对齐**: 把各系统的 Customer ID 统一。
        *   **标准落地**: 把“男/女”、“M/F”统一转码。
        *   **模型构建**: 按照 3NF (第三范式) 构建企业级逻辑模型。**注意**：这一层不面向具体报表设计，而是面向“企业全景”设计。

#### 第三层：数据服务层 (Service Layer)
*   **功能**: 面向业务场景组装数据。
*   **形式**: 
        *   **数据集市 (Data Mart)**: 比如“供应链集市”、“财经集市”。
        *   **数据服务 (API)**: 提供“查订单状态”、“查库存”的 Restful 接口。
*   **特点**: 这一层可以为了性能做冗余（宽表），采用维度建模 (Kimball)。

## 4.1.2 阿里数据架构设计与落地：中台驱动的“五层架构”

阿里架构是“OneData”体系的极致体现，核心目标是**复用**和**快**。

### 1. 核心设计理念
*   **OneData**: OneModel (统一模型), OneID (统一实体), OneService (统一服务)。
*   **公共层建设**: 极度强调构建“公共中间层”，让业务部门尽量基于中间层开发，而不是从头抽数。

### 2. 五层中台架构详解

#### 第一层：数据采集层 (ODS - Operational Data Store)
*   **工具**: DataWorks 数据集成。
*   **特点**: 实时 (Binlog) + 批量同步。支持异构数据源（MySQL, HBase, Log）。

#### 第二层：数据存储层 (Unified Storage)
*   **引擎**: MaxCompute (离线) + Hologres (实时) + OSS (非结构化)。
*   **策略**: 存储计算分离，低成本无限扩容。

#### 第三层：数据计算层 (Compute / OneModel) —— **核心**
阿里把这一层细分为：
*   **DWD (Data Warehouse Detail)**: 明细层。清洗后的一致性明细数据。
*   **DWS (Data Warehouse Summary)**: 汇总层。这是中台的精华。
        *   **轻度汇总**: 按“天/商户/商品”粒度汇总。
        *   **通用性**: 这里的指标必须是所有业务线通用的（如 GMV），而不是某个活动专用的。

#### 第四层：数据服务层 (OneService)
*   **理念**: **Service as a Product**。
*   **实现**: 通过 API 网关自动生成数据 API。业务端（淘宝 App、商家后台）通过 API 获取数据，毫秒级响应。

#### 第五层：数据应用层 (ADS - Application Data Store)
*   **内容**: 个性化的业务报表、推荐算法特征数据。
*   **原则**: 这一层的数据“用完即走”，不具备复用性。

---

> **对比总结**:
> *   **华为**像盖摩天大楼，地基（集成层）打得非常深，结构异常稳固，适合几十年不变的实体业务。
> *   **阿里**像搭乐高积木，强调模块（DWS 层）的通用性和拼接速度，适合瞬息万变的互联网业务。
