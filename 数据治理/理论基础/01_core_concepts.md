# 第一章：数据治理核心概念与理论框架

## 1.1 数据治理的定义、边界与核心价值

### 1.1.1 **[DAMA]** 国际对数据治理的权威界定
- **区分治理与管理**: 深入剖析 **[Data Governance]** (数据治理) 与 **[Data Management]** (数据管理) 的区别。
- **核心逻辑**: "治理是制定规则 (Setting Rules)，管理是执行规则 (Executing Rules)"。
- **[Decisions]**: 治理关注决策权和问责制；**[Execution]** 关注具体操作和实施。

### 1.1.2 数字化转型中的战略定位
- **定位转变**: 分析数据治理如何推动企业从 **[IT Cost Center]** (IT成本中心) 转向 **[Data Value Center]** (数据价值驱动中心)。
- **赋能业务**: 数据治理不再是单纯的管控，而是赋能业务创新与决策优化。

### 1.1.3 核心目标体系
详解确保数据资产的五大核心指标：
- **[Availability]** (可用性)
- **[Integrity]** (完整性)
- **[Consistency]** (一致性)
- **[Security]** (安全性)
- **[Compliance]** (合规性)
- **优先级**: 明确各目标的落地优先级与关联关系（如安全是底线，可用是前提）。
