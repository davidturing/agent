# 第二章 数据治理核心知识领域详解

## 2.1 数据架构管理
- **2.1.1 架构对齐逻辑**
    - 探讨数据架构如何承接业务架构需求，指导技术架构选型，实现业务、数据、技术三层架构的协同统一。
- **2.1.2 关键架构设计**
    - 主数据（MDM）架构的唯一性设计要点，参考数据（Reference Data）的标准化管理流程与应用规范。

## 2.2 数据质量管理
- **2.2.1 质量评估维度**
    - 深入讲解准确性、完整性、一致性、及时性、唯一性、有效性六大核心维度，明确各维度的量化评估方法。
- **2.2.2 归因与闭环**
    - 建立“发现问题-归因分析-根源治理-持续监控”的质量管理闭环机制，形成数据质量问题的长效解决路径。

## 2.3 数据安全与合规管理
- **2.3.1 安全技术体系**
    - 数据分级分类标准制定方法，以及访问控制（IAM）、静态/动态脱敏、同态加密等核心安全技术的应用场景与实施要点。
- **2.3.2 合规体系搭建**
    - 基于GDPR及中国《数据安全法》《个人信息保护法》等法律法规的合规性要求解读，构建企业数据合规管理框架。

## 2.4 其他关键知识领域
- **2.4.1 数据集成与存储**
    - ETL/ELT模式的技术选型依据，数据仓库与数据湖的存储策略差异及融合方案（湖仓一体）。
- **2.4.2 元数据管理**
    - 业务元数据（语义）与技术元数据（结构）的映射关系，元数据驱动的数据血缘分析理论与应用价值。
